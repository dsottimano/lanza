Admin menu architecture

How the EmDash admin sidebar is assembled — and therefore what you can and can't change.

The sidebar is built in three layers

The nav is generated by buildNavItems(manifest, userRole) inside @emdash-cms/admin. It builds one flat array, in this order:

LayerSourceYou can change it?
DashboardHardcodedNo
Your collections — one item per collection, using its labelmanifest.collections ← your seed/seed.jsonYes, fully
Media, Menus, Widgets, Sections, Content Types, Categories, Tags, Users, Plugins, Import, Settings, SecurityHardcoded arrayNo (config can't touch it)
Plugin admin pagesEach plugin's adminPages, appended lastAdd-only

The collections loop is the lever

This is the part you control. Every collection in your schema is pushed into the nav automatically:

function buildNavItems(manifest, userRole) {
  const items = [{ id: "dashboard", title: "Dashboard", to: "/", icon: SquaresFour }];

  for (const [name, config] of Object.entries(manifest.collections))
    items.push({
      id: `collection-${name}`,
      title: config.label,            // ← your label shows in the sidebar
      to: "/content/$collection",
      params: { collection: name },
      icon: FileText,
    });

  items.push(/* Media, Menus, Widgets, Sections, Content Types,
                 Categories, Tags, Users, Plugins, Import, Settings, Security */);
}

Define a listings collection and "Listings" appears. Define pizzas and "Pizzas" appears. No menu code involved — see Customizing the admin.

What's hardcoded (and why it matters)

The generic chrome below your collections is a fixed array baked into the admin bundle. Through supported config you cannot:

  • rename a core item (e.g. "Media Library" → "Photos"),
  • hide a core item (e.g. remove "Import" or "Sections"),
  • reorder core items,
  • change the app branding in the nav.

The only way to alter these is to fork/patch @emdash-cms/admin — which means every EmDash upgrade becomes a merge conflict. Not recommended for a handful of sites.

Taxonomy gotcha: the nav only ever adds two taxonomy items — category and tag — and they're hardcoded by name. A custom taxonomy (e.g. neighborhood, cuisine) still works, but it won't get its own sidebar entry from this build.
Source: buildNavItems in @emdash-cms/admin/dist/index.js · EmDash 0.16.1.
