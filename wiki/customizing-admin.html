Customizing the admin for a domain

Make the admin feel like a real-estate CMS, a pizza backend, or anything else — by modeling content, not by hacking the UI.

The mechanism

Define collections in seed/seed.json. EmDash generates, for each one, automatically:

  • a sidebar item (using your label),
  • list/table views and create/edit forms with whatever fields you declared,
  • full rich-text editing (Portable Text) for body fields — headings, lists, code, links, images,
  • search, drafts, validation, multi-user editing.

The "domain flavor" is entirely your collection names, labels, and fields.

Example: real-estate listings

// seed/seed.json (sketch)
{
  "collections": [
    {
      "name": "listings",
      "label": "Listings",
      "labelSingular": "Listing",
      "fields": [
        { "name": "title",   "type": "text" },
        { "name": "price",   "type": "number" },
        { "name": "beds",    "type": "number" },
        { "name": "baths",   "type": "number" },
        { "name": "address", "type": "text" },
        { "name": "gallery", "type": "image", "multiple": true },
        { "name": "details", "type": "portableText" }
      ]
    },
    { "name": "agents", "label": "Agents", "fields": [ /* ... */ ] }
  ]
}

Result: the sidebar shows Listings and Agents, each a complete CRUD admin.

Example: pizza ecom

Same pattern, different nouns: collections pizzas, toppings, orders → sidebar items Pizzas / Toppings / Orders.

What's free vs. what needs a fork

You want…Supported?
Domain content types in the menu (Listings, Pizzas, Orders)Yes — collections
Domain fields, validation, media, relationsYes — schema
Per-domain custom tools/dashboards in /adminPartial — plugin admin pages (Block Kit)
Rename / reorder / hide the generic chromeNo — fork only
Custom taxonomy as its own menu itemNo — only category/tag get nav slots
Fully white-labeled, restructured admin shellNo — fork only

Verdict

EmDash is a strong fit when customization means "model the domain as collections and fields" — the menu reshapes itself. It's a poor fit if the goal is a fully rebranded/restructured admin shell, because that chrome isn't config-exposed.
